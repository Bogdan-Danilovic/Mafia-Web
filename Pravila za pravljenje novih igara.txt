═══════════════════════════════════════════════
1. GAME_STANDARD.md
═══════════════════════════════════════════════

Dokument koji opisuje kako svaki novi game repo 
mora biti strukturiran da bi scripts/add-game.ts 
mogao automatski da ga integriše.

Mora sadržati:

A) Obavezna folder struktura:
   - Koji folderi moraju postojati
   - Koji fajlovi su obavezni
   - Koji su opcioni

B) game.config.json — kompletan standard:
   - Sva obavezna polja sa tipovima i opisom
   - Sva opcionalna polja
   - Primjer za Impostor igru
   - Primjer za Alias igru (hipotetički)

C) Pravila za komponente:
   - Kako se moraju zvati ekrani
   - Koji props moraju imati
   - Koji imports su zabranjeni (hardcoded putanje)
   - Koje importe sistem automatski adaptira

D) Pravila za TypeScript:
   - strict mode obavezan
   - zero any, zero ts-ignore
   - Koji tipovi se nasljeđuju iz Game Hub core.ts

E) Pravila za dizajn:
   - Design tokeni koji se moraju koristiti
   - Koje CSS varijable su dostupne
   - Font (Space Grotesk)
   - Animacije (Framer Motion, spring stiffness:300 damping:20)
   - Mobile-first, dvh ne vh

F) Pravila za Firestore:
   - Kako se mora zvati kolekcija
   - Obavezna polja u svakom dokumentu
   - connectedAlive < 3 graceful end (obavezno)

G) Pravila za rute:
   - Home: /games/{gameId}
   - Room: /games/{gameId}/room/[code]
   - Leave: uvijek na / (Hub)

H) Checklist prije pusha:
   - npm run build prolazi
   - tsc --noEmit prolazi
   - game.config.json validiran
   - Sve rute ispravne
